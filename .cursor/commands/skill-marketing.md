## The Feynman-Ogilvy-Rams Method

### ğŸ¨ Channel Rams First: Visual Before Verbal

Before writing anything, **design the visual experience**:

> "Good design is as little design as possible â€” less, but better."

**The Rams Process:**

1. **Identify the core visual metaphor** â€” What image/diagram/comparison will make this concept click?
2. **Design the hero visual** â€” What does the user see FIRST before reading?
3. **Plan visual hierarchy** â€” Eye path: Hero â†’ Problem visual â†’ Solution visual â†’ Proof â†’ CTA

**Rams Questions to Ask:**
- "What visual would make this concept obvious without reading?"

### ğŸ”¬ Channel Feynman Second

After planning visuals, apply **radical clarity** to copy:

> "If you can't explain it simply, you don't understand it well enough."

**The Feynman Process:**

1. **Name the concept** â€” What are we actually selling?
2. **Explain it to a child** â€” No jargon. What does it DO for someone?
3. **Identify gaps** â€” Where did you reach for buzzwords? That's where you don't understand.
4. **Go back to source** â€” Read the specs, understand the mechanism

**Feynman Questions to Ask:**
- "What is the *actual* mechanism here?" (Not the marketing spin â€” the real thing)
- "Why does this matter to someone who has real problems?"
- "What would I say if I couldn't use any industry jargon?"
- "Is this true? Could I defend it to a skeptical physicist?"

Once you have clarity, apply **persuasive craft**:

> "On average, five times as many people read the headline as read the body copy. When you have written your headline, you have spent eighty cents out of your dollar."

1. **The headline is 80% of your work** â€” Spend 80% of your effort there

**Ogilvy Questions to Ask:**
- "Would this headline make ME stop scrolling?"

- "Would this work as a direct mail piece where I pay per word?"

### The Triple Test

After designing any section, apply all three filters:

**Rams Filter:** "Is there a clear visual hierarchy? Would this look stunning in a screenshot?"

**Feynman Filter:** "Is this actually true? Could I explain this to a smart child without lying?"

**Ogilvy Filter:** "Would this make someone reach for their wallet? Does it have a big idea?"

If all three pass, ship it. If any fails, iterate.

## Body Copy Principles

### Feynman's Contribution: The Explanation Stack

When explaining any feature, layer from simple to complex:

```
Level 1: What does it DO? (A child understands)
Level 2: How does it WORK? (A smart layperson understands)
Level 3: Why is it BETTER? (A technical buyer understands)
Level 4: What's the MECHANISM? (An engineer is satisfied)
```

**Example: Explaining Capsules**

| Level | Copy |
|-------|------|
| 1. What | "A record of what your AI did" |
| 2. How | "Every decision, every file changed, every tool used â€” captured automatically" |
| 3. Why better | "Unlike chat logs, capsules are cryptographically signed. They prove what happened, not just what was said." |
| 4. Mechanism | "Post-quantum SPHINCS+ signatures make capsules tamper-evident and future-proof against quantum attacks" |

### Ogilvy's Contribution: The Proof Hierarchy

Every claim needs proof. Stack them from emotional to logical:

```
1. Emotional proof â€” How does this FEEL? (Stories, implied success)
2. Social proof â€” Who else believes this? (Testimonials, logos, numbers)
3. Logical proof â€” Does this make SENSE? (Comparisons, mechanisms)
4. Empirical proof â€” Can you SHOW me? (Demos, trials, evidence)
```

### ğŸ¯ Visual Hierarchy Rules

**The 3-Second Test:** A visitor should understand the page value in 3 seconds through visuals alone.

| Element | Visual Treatment |
|---------|------------------|
| **Primary Message** | 4xl-7xl headline, gradient text, above fold |
| **Social Proof** | Stats row with large numbers, subtle colors |
| **Key Features** | Icon + gradient background cards in grid |
| **Technical Detail** | Code windows, diagrams, monospace text |
| **CTA** | Full gradient button, shadow, glow, centered |

**Color Intensity by Importance:**

```
Most Important â†’ Full gradient (from-primary-500 to-accent-500)
Important â†’ Single solid color (text-primary-400)
Supporting â†’ Muted (text-slate-400)
Background â†’ Near invisible (bg-white/[0.02])
```

## Steps

### Phase 1: Visual Planning (Rams)

1. **Sketch the hero visual** â€” What image/diagram/comparison will make this concept click instantly?
2. **Plan visual sections** â€” Each section needs a focal visual element
3. **Choose the visual metaphor** â€” Before/after? Timeline? Code mockup? Architecture diagram?
4. **Select color palette** â€” Which gradient pairs for icons? What accent colors?
5. **Map interactions** â€” What animates on entrance? What responds to hover?

### Phase 2: Copy Development (Feynman + Ogilvy)

6. **Apply Feynman** â€” Understand the feature at mechanism level
7. **Find the big idea** â€” What's the Ogilvy hook?
8. **Write 10 headlines** â€” Spend 80% of effort here
9. **Test each headline** â€” Feynman: Is it true? Ogilvy: Does it sell?
10. **Build the explanation stack** â€” Simple to complex
11. **Add proof hierarchy** â€” Emotional to empirical

### Phase 3: Assembly

12. **Build sections** â€” Start with hero, work down
13. **Add micro-interactions** â€” Hover states, staggered animations
14. **Check against existing pages** â€” Match patterns in `@frontend/src/features/marketing/pages/`
15. **Run triple checklist** â€” Rams (visual) + Feynman (clarity) + Ogilvy (persuasion)
16. **Wire routing** â€” Add to App.tsx if new page

## The Triple Checklist

### âœ… Rams Filter (Visual)

- [ ] Does each section have a clear visual focal point?
- [ ] Is there a hero visualization (before/after, mockup, or diagram)?
- [ ] Do feature cards have gradient icons with hover effects?
- [ ] Is there at least one technical mockup (code window/terminal)?
- [ ] Are backgrounds layered with depth (gradients, blurs, patterns)?
- [ ] Do CTAs have animated gradients and glow effects?
- [ ] Are entrance animations staggered for visual rhythm?
- [ ] Would someone screenshot this page to share?
- [ ] Does the 3-second test pass? (Value clear from visuals alone)

### âœ… Feynman Filter (Clarity)

- [ ] Could I explain this to a smart 12-year-old?
- [ ] Have I removed all unnecessary jargon?
- [ ] Is every claim actually true and defensible?
- [ ] Do I understand the mechanism well enough to draw it?
- [ ] Would a skeptical engineer respect this?

### âœ… Ogilvy Filter (Persuasion)

- [ ] Is there a big idea, or just competent copy?
- [ ] Did I spend 80% of effort on the headline?
- [ ] Are my claims specific, not vague?
- [ ] Would this work as direct mail where I pay per word?
- [ ] Would this make ME stop scrolling and reach for my wallet?

## Anti-Patterns

### âŒ Rams Would Reject (Visual)

- **Wall-of-text sections** â€” If a section is >60% text, add a visual
- **Static, lifeless pages** â€” No hover effects, no animations
- **Flat, single-layer backgrounds** â€” Always layer depth
- **Icon-only feature lists** â€” Icons need gradient backgrounds
- **Generic stock imagery** â€” Use custom diagrams, mockups, or code
- **Unstyled code blocks** â€” Always use terminal mockups with window chrome
- **Missing visual hierarchy** â€” Everything same size = nothing important
- **No before/after** â€” Transformation should be VISIBLE, not just described
- **Boring CTAs** â€” Plain buttons without gradients, shadows, or glow

### âŒ Feynman Would Reject (Clarity)

- Jargon masquerading as explanation
- Claims you can't defend to a physicist
- Complexity added to seem impressive
- Vague mechanisms ("powered by AI")
- Hiding behind buzzwords you don't understand

### âŒ Ogilvy Would Reject (Persuasion)

- Headlines you could swap with any competitor
- Body copy without specific proof
- Adjectives doing the work of facts
- CTAs that don't compel action
- Copy that informs but doesn't sell
